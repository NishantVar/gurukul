# The Curriculum

This curriculum is built on the **Agentic Engineering Ladder** — eight levels that describe how people actually work with AI agents, from casual use to self-improving systems. Each level is a track. You move through them in order, and each one builds on what came before.

## At a Glance

| # | Track | What you'll learn |
|---|-------|-------------------|
| 1 | **Vibing** | Use a coding agent for real work just by asking naturally |
| 2 | **Operating** | Set up tools and environments that make your agent dramatically more capable |
| 3 | **Directing** | Write briefs that get consistent, verifiable results |
| 4 | **Delegating** | Split complex work across multiple agents |
| 5 | **Orchestrating** | Run a team of agents with defined roles and real handoffs |
| 6 | **Systemizing** | Build workflows that combine deterministic steps with AI |
| 7 | **Compounding** | Structure memory and context so past work carries forward |
| 8 | **Evolving** | Build a feedback loop that improves your whole system over time |

## Where Should You Aim?

**AI is a technology. If you know how to do what you need with it, you're an engineer.** Not a "prompt engineer," not an "AI enthusiast" — an engineer. The same way someone who knows how to build with electricity is an electrical engineer, someone who knows how to build with AI agents is an agentic engineer. The title isn't gatekept by a degree or a job posting. It's earned by capability.

**If you're not planning to work on production-level systems, aim for somewhere between levels 4 and 5.** That's the sweet spot where you can delegate real work to agents and coordinate them effectively — without needing to build the reusable infrastructure that levels 6+ require. Most people doing knowledge work, creative work, or personal projects will get enormous leverage from Delegating and Orchestrating without ever needing to Systemize.

## The Eight Tracks

### 1. Vibing

**Use a coding agent for real work just by asking naturally** — no special setup or expertise required.

This is where everyone starts. You talk to an agent in plain language, it does things for you, and you judge the results by whether they seem right. No configuration, no frameworks — just a conversation that produces real output.

### 2. Operating

**Set up tools and environments that make your agent dramatically more capable** — so you get better results without thinking harder.

The agent is the same, but *the setup around it changes everything*. Choose the right tools, connect the right systems, and let good defaults carry more of the work. Your results improve because the environment improves — not because you became an expert overnight.

### 3. Directing

**Write briefs that get consistent, verifiable results** — so you stop getting outputs that look right but aren't.

This is where you shift from hoping the agent understood to *knowing* it did. Define what success looks like, what constraints matter, and how to verify the result — before the work starts.

### 4. Delegating

**Split complex work across multiple agents** — so you can tackle bigger projects than any single agent can handle cleanly.

One agent has limits. Break work into bounded pieces, assign each piece to a separate agent, and bring the results back together. The work gets bigger without getting messier.

### 5. Orchestrating

**Run a team of agents with defined roles and real handoffs** — so complex, multi-part work gets done without you being the bottleneck.

This isn't just "more agents." You design a real division of labor — who does what, in what order, and how the pieces fit together. **The coordination becomes deliberate, not improvised.**

### 6. Systemizing

**Build workflows that combine deterministic steps with AI** — so you reduce the uncertainty in agent behavior and dramatically improve accuracy and reliability.

You stop relying on memory and luck. Good outcomes become the default because you encode your operating model into reusable workflows, checks, and guardrails that anyone could run.

### 7. Compounding

**Structure memory and context so past work carries forward** — so every task you do makes the next one better.

The system stops resetting between tasks. Decisions, patterns, and context accumulate into leverage. **Future work starts ahead** because past work is retrievable and trustworthy — not just stored somewhere.

### 8. Evolving

**Build a feedback loop that improves your whole system over time** — so it keeps getting better even when you're not actively working on it.

The final level. The system observes its own performance, evaluates what's working, and upgrades its own workflows, memory, and agent roles. **The ceiling keeps rising.**

## How the Tracks Connect

The ladder is sequential — **each level assumes you've internalized the ones before it.** You don't skip Directing to get to Delegating, because delegation without clear briefs produces chaos.

That said, **higher is not always better for every task.** A quick bug fix doesn't need an orchestrated agent team. The goal is range: knowing which level a task actually requires, and operating there cleanly.
